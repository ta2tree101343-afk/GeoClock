# ジオフェンス勤怠管理アプリ MVP要件定義

## 概要

工事現場の外国人労働者向けジオフェンス勤怠管理システム。

### 背景

- 工事現場の外国人労働者との言語コミュニケーションが困難
- 勤怠状況の把握が難しい
- 工事現場は基本屋外のため、GeoFenceによる位置検知が有効

### 用語集

| 用語 | 説明 |
| ------ | ------ |
| 勤務地 | Geofence（ジオフェンスエリア） |
| 管理者 | Admin Console（Web）を使用 |
| 労働者 | Mobile App（モバイルアプリ）を使用 |

---

## 確定要件

### 機能要件

#### 管理者（Admin Console）

- 勤務地の登録
- 労働者を勤務地へ登録
- 勤務地ごとの労働者の勤怠状況を確認

#### 労働者（Mobile App）

- ログイン機能（メール + パスワード）
- 管理者によって登録された勤務地を確認
- 勤務地内にinした場合に出勤、outした場合に退勤
- アプリがアクティブになったときに内部的に勤務地を登録
- バックグラウンドでの位置情報取得（ジオフェンス機能のため必須）

### 非機能要件

| 項目 | 内容 |
| ------ | ------ |
| 多言語対応 | 優先度低（後回し） |
| オフライン対応 | なし（想定しない） |
| 位置情報取得 | バックグラウンド必須 |
| ジオフェンス半径 | 最低100m |
| 複数勤務地 | 許可 |

### 認証

| 項目 | 内容 |
| ------ | ------ |
| アカウント作成 | 管理者が行う |
| ログイン方法 | メール + パスワード |
| 初回ログイン | パスワード変更必須 |
| パスワードリセット | 必要 |
| 将来対応 | Google/Microsoft SSO |
| 認証基盤 | AWS Cognito |

## 画面設計

### 労働者アプリ

#### 画面一覧

| 画面 | 概要 |
| ------ | ------ |
| スプラッシュ | トークン確認、自動遷移 |
| ログイン | メール + パスワード |
| パスワード変更 | 初回ログイン時（Cognito: NEW_PASSWORD_REQUIRED） |
| パスワードリセット | コード入力 + 新パスワード |
| ホーム | 勤務地ステータス一覧 + マップ（1画面統合） |

#### ログインフロー

```text
[スプラッシュ画面]
    ↓ トークン確認
    ↓ あり → [ホーム画面へ]
    ↓ なし
↓
[ログイン画面]
  - メールアドレス入力
  - パスワード入力
  - ログインボタン
  - 「パスワードを忘れた方」リンク
    ↓ 初回ログイン検知（Cognito: NEW_PASSWORD_REQUIRED）
[パスワード変更画面]
  - 新しいパスワード入力
  - 確認用パスワード入力
  - 変更ボタン
    ↓
[ホーム画面へ]
```

#### パスワードリセットフロー

```text
[ログイン画面]
    ↓ 「パスワードを忘れた方」タップ
[メールアドレス入力画面]
    ↓ 送信
[確認コード入力画面]
  - 確認コード入力
  - 新しいパスワード入力
    ↓
[ログイン画面へ戻る]
```

#### ホーム画面構成

```text
[ホーム画面]
  ┌─────────────────────────┐
  │ 勤務地ステータス一覧      │
  ├─────────────────────────┤
  │ 📍 新宿現場A             │
  │   出勤中 - 08:32 IN      │
  ├─────────────────────────┤
  │ 📍 渋谷現場B             │
  │   退勤 - 昨日 17:45 OUT  │
  ├─────────────────────────┤
  │ 📍 池袋現場C             │
  │   未出勤                 │
  └─────────────────────────┘
  ┌─────────────────────────┐
  │ 地図表示（メイン）        │
  │ - 現在地表示             │
  │ - 勤務地をピンで表示      │
  │ - ジオフェンス範囲を円で  │
  └─────────────────────────┘
```

#### バックグラウンド処理フロー

```text
[アプリ起動時]
    ↓
[位置情報パーミッション要求]
  - 「常に許可」が必要
    ↓
[通知パーミッション要求]
    ↓
[ジオフェンス登録]
  - APIから勤務地一覧取得
  - 各勤務地をジオフェンスとして登録
    ↓
[バックグラウンド監視開始]
    ↓
IN検知
[APIへ出勤記録送信]
[ローカルPush通知: 「新宿現場Aに出勤しました」]
    ↓
OUT検知
[APIへ退勤記録送信]
[ローカルPush通知: 「新宿現場Aから退勤しました」]
```

## データモデル設計

### Worker（労働者）

| フィールド | 型 | 説明 |
| ----------- | ----- | ------ |
| id | string (UUID) | PK、Cognito subと同一 |
| email | string | ユニーク |
| name | string | 表示名 |
| createdAt | timestamp | 作成日時 |
| updatedAt | timestamp | 更新日時 |

※ Admin（管理者）はCognito管理のみ、DBテーブル不要

### Geofence（勤務地）

| フィールド | 型 | 説明 |
| ----------- | ----- | ------ |
| id | string (UUID) | PK |
| name | string | 勤務地名 |
| latitude | number | 緯度 |
| longitude | number | 経度 |
| radius | number | 半径（メートル）、最低100m |
| address | string? | 住所（任意） |
| createdBy | string | 作成した管理者のUser ID |
| createdAt | timestamp | 作成日時 |
| updatedAt | timestamp | 更新日時 |

### WorkerGeofence（労働者と勤務地の紐付け）

| フィールド | 型 | 説明 |
| ----------- | ----- | ------ |
| id | string (UUID) | PK |
| workerId | string | FK → Worker |
| geofenceId | string | FK → Geofence |
| assignedAt | timestamp | 登録日時 |

※ 複数勤務地への登録を許可するため中間テーブル

### AttendanceLog（勤怠記録）

| フィールド | 型 | 説明 |
| ----------- | ----- | ------ |
| id | string (UUID) | PK |
| workerId | string | FK → Worker |
| geofenceId | string | FK → Geofence |
| type | literal | `"in"` / `"out"` |
| timestamp | timestamp | 打刻日時 |
| latitude | number | 打刻時の緯度 |
| longitude | number | 打刻時の経度 |
